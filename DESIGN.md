## 零、核心认知

```
meme coin 价值 = 叙事驱动的注意力定价

链上数据是结果，不是原因。
链上数据是验尸官，不是预言家。

执行铁律：link字段 → 推特传播链 → 盘面数据
不能先看链上决定"要不要查"，也不能因为链上脏就 pass。
脏也要列出来，让判断者自己决定。
```

本 skill 复用了两个已安装的 skill 作为数据采集层：

## 二、叙事分析子系统（★ 核心中的核心）

### 2.1 六类叙事分类体系

拿到一个币，**先归类，再套对应逻辑**。不同类型用不同的判断框架，不能混用。

---

#### A 型：事件驱动 — 已发生

**定义**：真实世界发生了某件事，币的叙事直接绑定该事件。

**典型案例**：某个名人发了条推、某公司出了个新闻、某政治人物说了句话

**判断框架（LLM 逐项分析）**：

| 维度 | 问题 | 评分权重 |
|------|------|---------|
| **时效性** | 事件距今多久？ | 0-24h 极强 / 1-3天 强 / 3-7天 中 / 7天+ 弱 |
| **媒体覆盖** | 主流媒体覆盖量？收视/阅读量是硬指标 | 高 |
| **传播 Hook** | 独立传播 hook 有几个？能否被反复引用？ | 高 |
| **命名关联** | 名字和事件的关联直觉明显吗？三秒能 get？ | 中 |
| **上限对标** | 同类事件之前的 meme 跑到多少 MC？ | 中 |

**核心风险**：事件新鲜度有衰减曲线，热度不会二次爆发，除非有第二波事件触发。

**致命陷阱**：事件被证伪 = 叙事瞬间死亡 = 立刻归零。

---

#### B 型：事件驱动 — 预期炒作

**定义**：尚未发生但预期会发生的事件，市场提前炒作。

**典型案例**：大选、政策公布、产品发布、名人即将做某事

**判断框架（LLM 逐项分析）**：

| 维度 | 问题 | 评分权重 |
|------|------|---------|
| **事件确定性** | 事件发生的概率？会不会被取消/延期？ | 极高 |
| **时间窗口** | 距离事件还有多久？1-2周最佳布局窗口 | 高 |
| **全球关注度** | 事件本身的全球关注度？是区域性还是全球性？ | 高 |
| **命名竞争** | 同事件有多少个盘在抢？命名空间拥挤度？ | 中 |
| **利好出尽风险** | 事件发生后是不是就该卖了？ | 高 |

**核心风险**：事件取消/延期 = 叙事证伪 = 归零。政治 meme 寿命 = 事件窗口期，窗口关闭就是归零，没有"反弹"。

**致命陷阱**：时区风险。重大事件通常在美东时间发布，亚洲交易员天然劣势。

---

#### C 型：文化梗型

**定义**：源自互联网文化、亚文化社区的梗，经过社区发酵后发币。

**典型案例**：PEPE、WIF、MOG、BONK

**判断框架（LLM 逐项分析）**：

| 维度 | 问题 | 评分权重 |
|------|------|---------|
| **梗新鲜度** | 第一次出现 vs 翻炒冷饭？ | 高 |
| **理解门槛** | 需要什么背景知识才能 get？越低越好 | 高 |
| **视觉符号** | 有没有强视觉符号？能做成表情包？ | 极高 |
| **二创空间** | 能不能被 remix/二次创作？ | 高 |
| **社区重叠** | 原生社区和 crypto 用户重叠度？ | 中 |
| **历史先例** | 同类文化梗历史上有没有成功案例？ | 中 |

**核心特征**：文化梗型是生命周期最长的 meme coin 类型（月级），因为文化梗本身有持续演化的能力。

**致命陷阱**：梗没有传播基因——有些梗在小圈子很火但无法破圈（如 PUNDU 熊猫）。

---

#### D 型：情感叙事型

**定义**：触发原始情感的叙事，让持有者产生情感共鸣和身份认同。

**典型案例**：保护弱小、逆袭故事、反叛精神（anti-VC/anti-establishment）

**判断框架（LLM 逐项分析）**：

| 维度 | 问题 | 评分权重 |
|------|------|---------|
| **情感类型** | 触发哪种原始情绪？（保护/逆袭/愤怒/搞笑） | 极高 |
| **情感载体** | 有没有具体的"人物/动物"作为情感载体？ | 高 |
| **跨文化共鸣** | 全球通用还是地域性？ | 高 |
| **首爆平台** | 在哪个平台首爆？TikTok > Twitter > Reddit > 新闻 | 中 |
| **争议性** | 有没有对立面？有争议的传播更快 | 中 |

**核心特征**：情感叙事传播速度极快（小时级），因为情绪驱动转发。但生命周期短（1-2天），情绪过了就没了。

**致命陷阱**：情感被消费完——故事讲完了，没有新的情感燃料。

---

#### E 型：技术包装型

**定义**：披着技术/概念外衣的 meme coin，本质上仍是 meme。

**典型案例**：AI Agent coin、DeFi meme、"有实际用途的 meme"

**判断框架（LLM 逐项分析）**：

| 维度 | 问题 | 评分权重 |
|------|------|---------|
| **底层叙事** | 剥掉技术包装，底层叙事是什么？ | 极高 |
| **时间距离** | 与核心技术事件的时间距离？ | 高 |
| **命名竞争** | 同一热点下有多少个技术包装盘？ | 高 |
| **交付压力** | 需要持续交付来维持信仰吗？ | 中 |
| **理解门槛** | 散户能不能理解？门槛越低越好 | 中 |

**核心风险**：技术类叙事需要持续交付来维持信仰，但 meme coin 玩家的注意力周期远短于开发周期。一旦进入"交付后"阶段，没有新的想象力空间，就是顶。

**经典案例**：AI16Z——开源后 fork 泛滥，叙事被稀释。散户不 care 代码，care 的是能不能在 Timeline 上看到被讨论。

---

#### F 型：纯 Meme 型

**定义**：不绑定事件、不绑定文化、不绑定技术，纯粹因为"好笑/好看/好记"而传播。

**典型案例**：DOGE（早期）、SPX6900、GIGA

**判断框架（LLM 逐项分析）**：

| 维度 | 问题 | 评分权重 |
|------|------|---------|
| **名字/视觉** | 名字够不够短？够不够顺口？视觉够不够强？ | 极高 |
| **传播直觉** | 0.5 秒能不能 get 到？越傻越传播 | 极高 |
| **口号/Slogan** | 有没有一句话口号？能不能降低传播成本？ | 高 |
| **历史先例** | 同类纯 meme 有没有成功先例？ | 中 |
| **随机性** | 纯 meme 型本质上不可预测 | — |

**核心特征**：这是最难分析的类型，因为成功几乎完全随机。最好的策略是：看到了、觉得好笑、小仓位赌一把、不执着。

### 2.2 叙事强度综合评分

每类叙事按各自的判断框架打分后，汇总为**叙事强度分（满分 100）**：

```
叙事强度 = 类型内各维度加权得分

不跨类型比较。A 型的 80 分和 C 型的 80 分意义不同。
A 型 80 分 = 事件很强但生命周期短
C 型 80 分 = 文化根基深但爆发慢
```

**附加修正因子（跨类型通用）**：

| 修正因子 | 加分/扣分 | 条件 |
|---------|----------|------|
| 视觉传播力 | +0~10 | 强视觉符号、能做表情包、48x48 可辨识 |
| 名字传播力 | +0~10 | 容易打字、顺口、3-5字母大写、无下划线无数字 |
| 口号传播力 | +0~5 | 有可复制的口号/slogan |
| 争议驱动 | +0~5 | 有对立面/争议，传播更快 |
| 跨文化壁垒 | -0~15 | 非英语叙事向全球传播的损耗 |

### 2.3 叙事生命周期判定

```
萌芽期 → 发酵期 → 爆发期 → 成熟期 → 衰退/重生

判定依据（LLM 综合判断）：
├── 萌芽期：概念刚提出，< 3 个 KOL 讨论，几乎无链上活动
├── 发酵期：KOL 开始讨论，社区成型，叙事框架被明确化
├── 爆发期：多 KOL 集中讨论，价格暴涨，FOMO 入场
├── 成熟期：大所上线，主流关注，叙事确立但增量有限
└── 衰退期：讨论量断崖，叙事疲劳，或被更强叙事替代
```

**不同叙事类型的生命周期差异**：

| 类型 | 萌芽→发酵 | 发酵→爆发 | 爆发→成熟 | 成熟→衰退 | 总寿命 |
|------|----------|----------|----------|----------|--------|
| A 事件已发生 | 分钟级 | 小时级 | 天级 | 天级 | 1-7天 |
| B 事件预期 | 天级 | 天级 | 天级 | 天级 | 3-14天 |
| C 文化梗 | 天-周级 | 天级 | 周级 | 周-月级 | 月级 |
| D 情感叙事 | 小时级 | 小时级 | 天级 | 天级 | 1-3天 |
| E 技术包装 | 天级 | 天级 | 周级 | 周级 | 周-月级 |
| F 纯Meme | 随机 | 随机 | 随机 | 随机 | 随机 |

### 2.4 叙事护城河分析

**核心问题**：同一个叙事下如果有多个盘，哪个盘有独立传播引擎？

**同叙事吸血判断**：

```
同一叙事下的竞争分析：
├── 谁先发的？（先发优势）
├── 谁的传播链更强？（KOL 层级 + 传播量级）
├── 谁的名字/ticker 更直觉？（传播效率差异）
├── 谁的视觉更强？（识别度差异）
├── 谁的社区更活跃？（讨论质量 vs 刷量）
└── 谁有独立传播引擎？（不依赖同叙事其他盘的流量）
```

**独立传播引擎 = 不依赖"XX 概念"这个标签，而是有自己的传播动力。**

例：同样是"AI meme"，$GOAT 有独立的"AI 发币"故事线，不依赖"AI meme 赛道"这个标签。而大部分 AI meme coin 只是蹭 AI 热度，没有自己的故事。

### 2.5 注意力定价模型

```
当前 MC < 注意力即将流入的价值 = 定价错误 = 买入区间
```

**步骤 1：注意力估值**

| 维度 | 估算方法 |
|------|---------|
| 源头影响力 | 首发者粉丝量 × 互动率 |
| 传播层数 | 源头 → 大V → 小V → 社区，有几层？ |
| 传播量级 | 百万级 views + 大V自发接力 = 强 / 十万级 = 中 / 小KOL自嗨 = 弱 |
| 持续力 | 一次性事件 vs 持续发酵？有没有第二波催化剂？ |
| 争议性 | 有对立面/争议 = 传播加速系数 1.5-2x |

**步骤 2：找对标 + 打折**

```
1. 找到同类叙事的历史天花板 MC
2. 对比当前叙事的权重差异：
   - 源头权威性折扣
   - 传播层级折扣
   - 持续力折扣
   - 竞争密度折扣
3. 得出预期天花板 MC
```

**步骤 3：对照当前 MC**

| 关系 | 判断 |
|------|------|
| 当前 MC << 预期天花板 | ✅ 定价错误，有空间 |
| 当前 MC ≈ 预期天花板 | ⚠️ 性价比一般 |
| 当前 MC >> 预期天花板 | ❌ 已透支 |

## 三、传播链分析子系统

### 3.1 CT 传播六层模型 (L0-L6)

```
L0: 源头层 — 4chan/Reddit/Discord私群/TG alpha群
    ↓ 私域发酵 30min-2h
L1: 侦察层 — DexScreener猎手/GMGN聪明钱/Birdeye异动
    ↓ CA在小群扩散
L2: 早期传播层 ★ 最优风险收益比 ★
    ├─ 小KOL (<5k粉) 发帖
    ├─ CT出现原生讨论
    └─ holder 200-500
    ↓ 叙事被提炼和包装
L3: 叙事放大层 — 中型KOL (5k-50k粉) 跟进
    ├─ "下一个$XXX"类帖子出现
    └─ meme图开始流传
    ↓ FOMO蔓延
L4: 引爆层 — 大KOL (50k-500k粉) 发帖
    ├─ 多KOL集中讨论
    └─ MC $100K→$1M+
    ↓ 传播指数增长
L5: FOMO层 ★ 接盘侠区间 ★
    ├─ 头部KOL (500k+) 跟进
    └─ DexScreener Trending #1
    ↓ 讨论量峰值后衰减
L6: 破圈/衰退层 — TikTok/Instagram 或 讨论量断崖
```

**Skill 的主要作用区间：L2-L3**。此时数据已可获取、LLM 有足够时间分析、风险收益比最优。

### 3.2 Link 字段拆解（第一步，最关键）

GMGN 返回的 link 字段分四种情况：

| 情况 | 示例 | 处理方式 |
|------|------|---------|
| ① 推文路径 | "0xScimpy/status/2053306587721252940" | 提取 tweet_id → 获取推文详情 |
| ② 正常用户名 | "ScamAltmanCoin" | 查该用户推文/主页 |
| ③ 无厘头链接 | x.com/i/trending/xxx、新闻链接 | 必须打开看，藏着真正叙事来源 |
| ④ 空值 | — | 标注"无推特"，继续查其他维度 |

> link 字段是叙事的钥匙。跳过一个字段就可能 miss 一个 20x。

### 3.3 传播链四大核心问题

**1. 源头是谁？**
- 推文是个人号还是官媒？
- 粉丝量级：K (<10K) / 万级 (10-100K) / 十万级 (100K-1M) / 百万级 (1M+)

**2. 传播力如何？**
- 互动量：likes / RTs / replies / views（views 是最真实指标）
- views/followers 比值：>1 说明出圈了
- 有没有被大V接力转发？接力层数？
- 传播量级：百万级 views + 大V自发接力 = 强叙事

**3. 有没有争议/对立面？**
- 性别对立 / 政治立场 / 行业黑幕 — 有争议传播更快
- 评论区是支持还是骂？骂也是传播

**4. 传播的持续性？**
- 一次性推文还是持续讨论？
- 有没有媒体跟进？传统媒体介入 = 叙事升级信号
- 第二波催化剂预期？

### 3.4 KOL 传播图谱

**KOL 分类**：

| 类型 | 粉丝量 | 特征 | 可信度 | 对传播的价值 |
|------|--------|------|--------|------------|
| 源头发布者 | 不限 | 最早发帖的人 | 看情况 | 极高 |
| 侦察型 | <5k | L1-L2 层信息获取，独立判断 | 高 | 高 |
| 放大型 | 5k-50k | 有研究框架，个人风格 | 中高 | 高 |
| 引爆型 | 50k-500k | 影响力大，可能付费 | 中 | 中（入场时机差） |
| 头部 | 500k+ | 喊单效应强，滞后严重 | 低 | 低（入场时机差） |
| 喊单号 | 任意 | 模板化、删失败帖 | 极低 | 负 |
| 付费推广号 | 任意 | 多号同时推、语言一致 | 零 | 负 |

**付费推广识别**：
- 多个中型 KOL 在 30 分钟内集中发帖推同一项目
- 不同 KOL 用相同的叙事框架和关键词
- KOL 钱包在发帖前大量买入
- 内容空洞："什么都说了又什么都没说"

**Caller 互割识别**：
- KOL A 和 KOL B 频繁互动但从不讨论对方失败推荐
- KOL 发帖 5-15 分钟后出现大额卖单
- 发帖后 holder 增加但 MC 不涨反跌
- 多个大钱包在 KOL 发帖后同时卖出

### 3.5 传播规律

**传播力公式**：
```
传播力 = 文化共鸣度(0.40) + 争议性(0.25) + 视觉可传播性(0.20) + 叙事简洁度(0.15)
```

**五大规律**：
1. 越"傻"越传播 — 0.5 秒能 get > 需要思考 3 秒
2. 有争议的传播快 3-5 倍 — 争论 = 算法推流
3. 视觉 meme 比文字快 — 一张好图 = 10 条推文
4. "我们 vs 他们"天然鼓励分享
5. 可二创的 meme 生命周期更长

### 3.6 跨文化传播追踪

```
英文 CT → 中文 CT:   ~40%（需要"翻译"和重新包装）
英文 CT → 韩文 CT:   ~30%（需要本土化）
英文 CT → 日文 CT:   ~15%（文化隔阂大）
中文 CT → 英文 CT:   ~5%（几乎不可能反向传播）
```

**关键信号**：当 meme 从英文 CT 传到中文 CT 时，英文 CT 早期参与者通常已在考虑出货。

## 四、阶段分析子系统

### 4.1 六阶段模型

| 阶段 | 时间 | 核心问题 | 决策 |
|------|------|---------|------|
| 内盘早期 | 发射后 0-5min | 能不能毕业？ | 快速过滤，不决策 |
| 内盘中段 | 5min-毕业 | 毕业概率多大？ | 观察，准备毕业策略 |
| **毕业窗口** | **毕业前后 5min** | **持有还是卖？** | **关键决策节点** |
| 外盘早期 | 毕业后 5-30min | 会不会二次爆发？ | 入场/观望决策 |
| 外盘趋势 | 30min-6h | 趋势能否持续？ | 持仓管理 |
| 外盘后期 | 6-12h | 是否退出？ | 退出/减仓决策 |

### 4.2 毕业窗口专项分析

**三种毕业驱动力**：

| 驱动力 | 特征 | 毕业后走势预测 |
|--------|------|--------------|
| 自然买盘推动 | 买盘分散、社区有讨论、dev 未出货 | 大概率继续涨 |
| 被硬拉毕业 | 1-3 个钱包在拉、社区死寂 | 毕业即巅峰 |
| Dev 自己拉 | Dev 钱包在最后阶段大笔买入 | 短期可能还有一波，风险极高 |

**毕业窗口决策规则**：
```
毕业前 3 分钟内：
  买入地址 > 20 + 每笔 < $500 + dev 持仓稳定 → 持有/买入
  买入地址 < 5 + 单笔 > $1000 + dev 在卖 → 卖出/不碰
```

### 4.3 毕业后走势分类

| 走势 | 比例 | 关键特征 |
|------|------|---------|
| 毕业即巅峰 | 55-60% | 毕业放巨量后断崖缩量，前10出货 |
| 震荡阴跌 | 20-25% | 无人关注，慢慢归零 |
| **二次爆发** | **10-15%** | 回调30-50%后企稳，holder翻倍，KOL毕业后跟进 |
| 持续上涨 | 3-5% | 几乎不回调，强力叙事支撑 |

**二次爆发币的共同特征**：
1. 回调幅度 30-50%（太浅=筹码集中，太深=共识散了）
2. holder 毕业后 30 分钟内翻倍
3. DexScreener 评论有实质性讨论（非机器人刷 🚀🌕）
4. KOL 在毕业后（非毕业前）发推跟进
5. 回调缩量、反弹放量
6. Twitter 出现新用户自发讨论

## 七、态势判断与输出规范

### 7.1 态势判断类型

不是"建议买入/卖出"，而是：

- **"龙头已锁定"** — 叙事主导地位确立，同类难以撼动
- **"独立叙事空间"** — 有独立传播引擎，不受同叙事吸血
- **"跟风盘"** — 没有独立传播引擎，依赖母叙事流量
- **"预期天花板 $X"** — 基于注意力定价的预期上限
- **"叙事无升级空间"** — 故事讲完了，没有第二波催化剂
- **"等待第二波催化剂"** — 叙事有续集潜力，但需要新事件触发
- **"叙事被证伪"** — 核心故事被证明是假的，立刻清仓
- **"叙事被更强替代"** — 同赛道出现更强竞争者

### 7.2 研报输出结构

```
第一句：核心叙事钩子 — 一话说清"为什么能起来"
  ↓
叙事拆解：
  · 叙事类型（A/B/C/D/E/F）+ 阶段判定
  · 叙事强度评分 + 各维度得分
  · 来源权威性（谁发的、多少粉、互动数据）
  · 传播逻辑（源头→接力→量级、大V接力层数）
  · 情绪/争议驱动（人群共鸣点）
  · 护城河分析（同叙事竞争格局）
  ↓
传播链分析：
  · 当前传播层级（L0-L6）
  · KOL 图谱（谁在推、自然/付费/caller互割）
  · 跨文化传播状态
  ↓
注意力定价：
  · 对标项目 + 天花板 MC
  · 折扣因子 + 预期 MC
  · 当前 MC vs 预期 MC
  ↓
数据汇总（盘面数字、链上结构事实）
  ↓
安全一票否决检查结果
  ↓
一句话态势判断
```

### 7.3 格式规范

- 金额用 K/M：$1.2K、$120K、$1.2M
- CA 纯文本，不发链接
- 时间用相对时间 + CST
- 不用"首先/其次/最后"
- 善用**加粗**、分割线区分层级

## 八、反直觉经验与经典错误

### 8.1 五大反直觉经验

1. **越脏的盘越能涨** — 庄家有筹码才能拉盘，完全公平发射 = 没人负责拉盘
2. **Bot 率高是好事** — 做市商在铺流动性；要警惕的是 bot 率突然下降
3. **负面 FUD 是最好的买入信号** — 制造注意力 + 洗纸手 = 筹码集中
4. **TG/Discord 热度是反向指标** — 刷屏越快越危险，大户不在里面聊天
5. **CEX 上币往往是顶部** — 只有 Binance 现货值得持有

### 8.2 六大经典错误

1. **被链上数据挡住最重要的信息** — Bot/Bundler 再高也不是 pass 理由
2. **没看 link 字段** — 必须逐条查 twitter_username 和 website
3. **同叙事吸血误判** — 要评估哪个盘有独立传播引擎
4. **用价格否定叙事** — "涨太多了"不是合理判断
5. **分不清"工作室建仓"和"caller互割"** — 都是高 bot 率但性质不同
6. **一次采集不把传播链拉全** — 链条断了就等于白查

## 十一、核心差异化价值

> 这个 skill 不做喊单 bot 的事。
>
> 它的核心价值是：
> 1. **把信息变成信号** — 从噪音中提取有意义的叙事信号
> 2. **把直觉变成框架** — 六类叙事 × 六阶段 × 六层传播 = 可执行的判断体系
> 3. **把情绪变成纪律** — 结构化的叙事层面止损条件
> 4. **做市面上没有的事** — KOL 传播图谱 + 传播层级判定 + 注意力定价，没有任何工具在做
>
> 最终产出不是"买/卖"，而是"叙事是什么、传播到哪了、注意力值多少钱、当前处于什么态势"。

## 十二、数据采集 SOP（标准操作流程）

### 12.1 总流程

```
用户提供 CA 地址
    │
    ├─ Step 1: 并行采集（GMGN + Agent-Reach 同时跑）
    │   ├─ GMGN: token info（代币信息 + 安全 + 持仓 + link字段）
    │   └─ GMGN: portfolio created-tokens（Dev 历史）
    │
    ├─ Step 2: 解析 link 字段 → Agent-Reach 查叙事源头
    │   ├─ 情况① 推文路径 → Agent-Reach 读推文 + 查传播链
    │   ├─ 情况② 用户名 → Agent-Reach 查用户主页/推文
    │   ├─ 情况③ 链接 → Agent-Reach 读网页内容
    │   └─ 情况④ 空值 → 标注，继续
    │
    ├─ Step 3: 传播链深度分析（Agent-Reach）
    │   ├─ Twitter 搜索代币名称/ticker
    │   ├─ 全网搜索（Exa）相关新闻/媒体报道
    │   ├─ Reddit/B站/小红书 跨平台检测
    │   └─ 构建 KOL 传播图谱 + 传播层级判定
    │
    └─ Step 4: LLM 综合分析（本 Skill 核心）
        ├─ 叙事分类 + 强度评分
        ├─ 注意力定价
        ├─ 安全一票否决
        ├─ 叙事→资金验证
        └─ 态势判断 + 输出研报
```

### 12.2 Step 1 详细：GMGN 数据采集

```bash
# 并行执行以下查询：

# 1.1 代币基础信息（最重要，包含 link 字段）
gmgn-cli token info --chain sol --address <CA>

# 1.2 Dev 历史
# 先从 token info 中提取 dev 钱包地址，然后：
gmgn-cli portfolio created-tokens --chain sol --wallet <DEV_WALLET>

# 1.3 聪明钱/KOL 持仓（可选，增强分析）
gmgn-cli track smartmoney --chain sol --limit 20
gmgn-cli track kol --chain sol --side buy --limit 20
```

**从 GMGN token info 中提取的关键字段**：

| 字段 | 用途 | 对应分析环节 |
|------|------|------------|
| `link.twitter_username` | ★ 叙事钥匙 | link 字段拆解 → Agent-Reach |
| `link.website` | ★ 叙事钥匙 | link 字段拆解 → Agent-Reach |
| `price` / `market_cap` / `ath` | 盘面数据 | 注意力定价、阶段判定 |
| `creation_timestamp` | 创建时间（UTC→CST） | 阶段判定、新鲜度 |
| `holders_count` | 持币地址数 | 筹码结构 |
| `top_10_holder_rate` | Top 10 持仓占比 | 筹码结构 |
| `dev_hold_rate` | Dev 持仓占比 | Dev 行为分析 |
| `bundler_trader_amount_rate` | Bundler 占比 | 筹码结构 |
| `rat_trader_amount_rate` | 老鼠仓比率 | 筹码结构 |
| `sniper_count` | 狙击手数量 | 筹码结构 |
| `smart_degen_count` | 聪明钱数量 | 叙事→资金验证 |
| `is_on_curve` | 是否还在 bonding curve | 阶段判定 |
| `rug_risk_score` | Rug 风险评分 | 安全一票否决 |
| `honeypot_detected` | 蜜罐检测 | 安全一票否决 |
| `mint_authority_revoked` | Mint 权限是否放弃 | 安全一票否决 |
| `freeze_authority_revoked` | Freeze 权限是否放弃 | 安全一票否决 |

### 12.3 Step 2 详细：Link 字段拆解 + Agent-Reach

**情况①：推文路径**
```
GMGN 返回 twitter_username = "0xScimpy/status/2053306587721252940"
→ 提取 tweet_id = "2053306587721252940"
→ Agent-Reach: 读这条推文，获取：
   - 推文内容（叙事的原始文本）
   - 作者信息（粉丝量、是否官媒）
   - 互动数据（likes/RTs/replies/views）
   - 评论区内容（判断讨论质量）
```

**情况②：正常用户名**
```
GMGN 返回 twitter_username = "ScamAltmanCoin"
→ Agent-Reach: 查该用户推文/主页，获取：
   - 最近的推文内容
   - 账号年龄
   - 粉丝量
   - 发推频率
```

**情况③：无厘头链接**
```
GMGN 返回 website = "https://www.dailymail.co.uk/news/xxx"
→ Agent-Reach: Jina Reader 读取网页内容
→ 从中提取叙事来源
```

**情况④：空值**
```
→ 标注"无推特/网站内容"
→ 跳过传播链分析中的 Twitter 部分
→ 依赖 Agent-Reach 的全网搜索 + 跨平台检测来弥补
```

### 12.4 Step 3 详细：传播链深度分析

```bash
# 3.1 Twitter 搜索（Agent-Reach）
# 搜索代币名称、ticker、CA 地址
"帮我搜 Twitter 上关于 <TOKEN_NAME> 和 <TICKER> 的讨论"

# 3.2 全网搜索（Agent-Reach Exa）
"帮我全网搜索 <TOKEN_NAME> 相关的新闻报道"

# 3.3 跨平台检测（Agent-Reach）
"帮我看看 Reddit 上有没有关于 <TOKEN_NAME> 的讨论"
"帮我看看 B站/小红书有没有 <TOKEN_NAME> 相关内容"

# 3.4 KOL 传播图谱（Agent-Reach + LLM）
# 基于以上数据，LLM 构建：
# - 谁第一个发了这个币？
# - 哪些 KOL 转推/讨论了？
# - 传播路径是自然的还是付费的？
# - 当前处于 L0-L6 的哪一层？
```

### 12.5 数据采集耗时与并行策略

```
全程控制在 2-3 轮工具调用内，不来回确认：

第 1 轮（并行，~30s）：
  ├── GMGN: token info
  ├── GMGN: Dev 历史
  └── GMGN: 聪明钱/KOL 持仓

第 2 轮（依赖第1轮的 link 字段，并行，~30s）：
  ├── Agent-Reach: 读原始推文/网页
  ├── Agent-Reach: Twitter 搜索
  └── Agent-Reach: 全网搜索

第 3 轮（LLM 综合分析，~30-60s）：
  └── 叙事分类 + 传播链 + 定价 + 安全 + 态势 → 输出研报

总耗时：~2 分钟
```

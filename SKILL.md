# Meme Coin 投研分析

输入一个 CA（合约地址），输出一份固定格式的投研报告。

## 触发方式

```
/analyze <CA>     → 完整投研报告
/quick <CA>       → 快速定性（只出叙事类型 + 安全 + 态势一句话）
```

自然语言触发：`分析这个meme`、`帮我做一下投研`、`看看这个币怎么样`，后跟 CA 地址。

---

## 核心认知

```
meme coin 价值 = 叙事驱动的注意力定价
链上数据是结果，不是原因
执行铁律：link字段 → 传播链 → 盘面数据
不能因为链上脏就 pass，脏也要列出来
```

---

## 执行流程

收到 CA 后，严格按以下 3 轮执行，不跳步、不颠倒、不来回确认。

### 第 1 轮：并行数据采集

**同时**执行以下查询：

```
① GMGN 代币信息（Skill: gmgn）
   调用 gmgn skill 的 token info 功能，查询代币基础信息。
   必须提取以下字段：
   - link 字段（twitter_username, website）★ 最关键
   - price, market_cap, ath
   - creation_timestamp（UTC时间，需要 +8h 转北京时间）
   - holders_count, top_10_holder_rate
   - dev_hold_rate, bundler_trader_amount_rate
   - rat_trader_amount_rate, sniper_count
   - smart_degen_count, is_on_curve
   - rug_risk_score, honeypot_detected
   - mint_authority_revoked, freeze_authority_revoked

② GMGN Dev 历史（Skill: gmgn）
   从 token info 结果中提取 dev 钱包地址后，调用 gmgn 查 Dev 创建过的所有代币。
   提取：历史币数量、毕业率、rug率、平均存活时间、最高MC

③ Agent-Reach 读原始叙事（Skill: agent-reach）
   根据 token info 返回的 link 字段，分四种情况处理：
   
   情况① 推文路径（如 "用户名/status/1234567890"）
     → 提取 tweet_id，用 agent-reach 读这条推文
     → 获取：推文内容、作者粉丝量、likes/RTs/replies/views、评论区
   
   情况② 正常用户名（如 "ScamAltmanCoin"）
     → 用 agent-reach 查该用户主页和最近推文
     → 获取：账号年龄、粉丝量、发推频率、最近内容
   
   情况③ 网页链接（如新闻网站、x.com trending 链接）
     → 用 agent-reach 的网页阅读功能打开链接
     → 提取真正的叙事来源
   
   情况④ 空值
     → 标注"无推特/网站"，跳过此步，但后续必须用 agent-reach 全网搜索弥补

   ⚠️ link 字段是叙事的钥匙，跳过一个字段就可能 miss 一个 20x。
```

### 第 2 轮：传播链深度分析

```
④ Agent-Reach Twitter 搜索
   用 agent-reach 在 Twitter 上搜索代币名称和 ticker。
   回答以下问题：
   - 讨论量有多少？讨论趋势是上升还是下降？
   - 有哪些 KOL 在讨论？（记录粉丝量级、发帖时间、内容质量）
   - 讨论是自然传播还是付费推广？（多 KOL 在 30min 内集中发帖 = 付费）
   - 有没有形成争议/对立面？（有争议传播更快）
   - 评论区的质量如何？（真人在讨论 vs 机器人刷 LFG）

⑤ Agent-Reach 全网搜索
   用 agent-reach 的 Exa 语义搜索，搜索代币名称 + 相关关键词。
   获取：是否有传统媒体报道、是否有新闻覆盖。

⑥ Agent-Reach 跨平台检测（按需）
   如果代币已存在一段时间（>6h），检查：
   - Reddit: r/CryptoMoonShots 等板块是否有讨论
   - 中文社区: B站/小红书是否出现相关内容
   - 判断是否已跨文化传播
```

### 第 3 轮：LLM 综合分析 → 输出固定格式研报

基于前两轮的所有数据，LLM 按以下框架分析并输出报告。**严禁跳过任何章节。**

---

## 分析框架（LLM 执行）

### 一、安全一票否决

逐条检查以下 6 条。**任一命中 → 直接标记"安全不通过"，跳过后续分析，仅输出安全检查章节 + 不通过原因**：

```
☐ 1. LP 未锁定/未销毁 — pump.fun 毕业币自动满足，非 pump.fun 必须检查
☐ 2. Mint Authority 未 revoke — 一票否决，无例外
☐ 3. Freeze Authority 未 revoke — 一票否决，无例外
☐ 4. 有未披露的 Transfer Fee 修改权限 — 看起来没税，Dev 可随时改成 99%
☐ 5. 前 3 地址（排除CEX/DEX/销毁地址）持仓 > 30% 且未锁定
☐ 6. Dev 在创建后 24h 内卖出 > 初始持仓 10%
```

### 二、叙事分类与判断

先判断属于哪种类型，再套对应框架分析。**一个币只能属于一种类型**：

**A. 事件驱动 — 已发生**
- 事件距今多久？（0-24h 极强 / 1-3天 强 / 3-7天 中 / 7天+ 弱）
- 主流媒体覆盖量？收视/阅读量是硬指标
- 独立传播 hook 有几个？
- 名字和事件的关联直觉明显吗？
- 上限对标：同类事件之前的 meme 跑到多少 MC

**B. 事件驱动 — 预期炒作**
- 事件发生的确定性有多高？会不会取消/延期？
- 距离事件还有多久？1-2周是最佳布局窗口
- 事件本身的全球关注度？
- 同事件有多少个盘在竞争命名空间？
- 有没有"利好出尽"风险？

**C. 文化梗型**
- 梗在原生社区的新鲜度？（第一次出现 vs 翻炒冷饭）
- 理解门槛高低？（越低越好）
- 有没有强视觉符号？（能做成表情包吗？缩到 48x48 还能认吗？）
- 二创空间？（能被 remix 吗？）
- 原生社区和 crypto 用户重叠度？
- 同类文化梗历史上有没有成功先例？

**D. 情感叙事型**
- 触发哪种原始情绪？（保护弱小 / 逆袭 / 愤怒 / 搞笑）
- 有没有具体的"人物/动物"作为情感载体？
- 跨文化共鸣度？（全球通用 vs 地域性）
- 在哪个平台首爆？（TikTok > Twitter > Reddit > 新闻）

**E. 技术包装型**
- 剥掉技术包装，底层叙事是什么？
- 与核心技术事件的时间距离？
- 同一热点下命名竞争密度？
- 需要持续交付来维持信仰吗？（散户不 care 代码，care 的是 Timeline 上有没有讨论）

**F. 纯 Meme 型**
- 同类历史上有没有成功先例？
- 视觉符号够不够强？
- 名字够短够顺口？ticker 念出来顺不顺？
- 0.5 秒能不能 get 到？

### 三、传播链分析

1. **源头判定**：谁第一个发的？粉丝量级：K (<10K) / 万级 (10-100K) / 十万级 (100K-1M) / 百万级 (1M+)
2. **传播层级**：当前处于 L0-L6 哪一层？
   - L0-L1: 私域发酵（4chan/Reddit/Discord/链上监控）
   - L2: 早期传播 — 小KOL发帖，CT原生讨论 ★ 最优风险收益比
   - L3: 叙事放大 — 中型KOL跟进，"下一个$XXX"帖子出现
   - L4: 引爆 — 大KOL集中讨论，MC $100K→$1M+
   - L5: FOMO — 头部KOL跟进，DexScreener Trending #1 ⚠ 接盘风险
   - L6: 破圈/衰退 — TikTok/Instagram 或讨论量断崖
3. **传播量级**：views 总量、views/followers 比值（>1 = 出圈）、大V接力层数
4. **传播质量**：自然传播 vs 付费推广 vs caller互割
5. **争议驱动**：有无对立面/争议（加速传播 1.5-2x）
6. **持续性**：一次性推文 vs 持续讨论？有无第二波催化剂？

### 四、阶段判定

| 阶段 | 判定条件 |
|------|---------|
| 内盘早期 | is_on_curve=true, MC < $30K |
| 内盘中段 | is_on_curve=true, MC $30K-$69K |
| 毕业窗口 | 刚毕业（< 5min）或即将毕业（MC 接近 $69K） |
| 外盘早期 | 已毕业，毕业后 5-30min |
| 外盘趋势 | 毕业后 30min-6h |
| 外盘后期 | 毕业后 6h+ |

**毕业窗口额外判断**：
- 自然买盘推动毕业（买盘分散 + 社区讨论 + dev 未出货）→ 毕业后大概率继续涨
- 被硬拉毕业（1-3个钱包在拉 + 社区死寂）→ 毕业即巅峰
- Dev 自己拉毕业（Dev 钱包最后阶段大笔买入）→ 短期可能有 DexScreener 流量，但风险极高

### 五、注意力定价

```
步骤1: 找对标 — 同类叙事的历史天花板 MC 是多少？
步骤2: 打折 — 当前叙事的源头权威性、传播层级、持续力、竞争密度各打多少折扣？
步骤3: 对照 — 当前 MC 和预期天花板 MC 的关系：
  - 当前 MC << 预期 → 定价错误，有空间
  - 当前 MC ≈ 预期 → 性价比一般
  - 当前 MC >> 预期 → 已透支
```

### 六、筹码与验证

**脏盘量化**（所有脏东西加在一起）：
- Dev 持仓 ___%（可接受 ≤5%，致命 >10%）
- 狙击手/Bundler ___%（可接受 ≤20%，致命 >30%）
- 老鼠仓 ___%（可接受 ≤3%，致命 >10%）
- Top 10（排除已知地址后）___%（可接受 ≤30%，致命 >50%）
- **总量**：___%（总原则：不超过 30-35%）

**叙事→资金验证**：
- 正向信号：新地址净买入>0、持币时间增加、聪明钱在叙事发酵后 2-6h 入场、KOL 喊单后链上持续净买入
- 负面信号：喊单激增但交易量不涨、热度高但 holder 不涨、价格微涨但持币地址减少

## 格式规范

- 金额用 K/M/B：$12.3K、$1.2M、$45M
- CA 纯文本，不发链接
- 时间用相对时间 + 北京时间（CST）："3小时前（北京时间 14:30）"
- 不用"首先/其次/最后/此外"等过渡词
- 善用加粗、表格、分割线区分层级
- 所有百分比精确到小数点后一位
- 缺失数据标注"数据不足"，不编造、不留空
- **安全不通过时，只输出安全检查章节，不输出后续章节**

## 反直觉提醒（分析时必须记住）

1. 链上脏不是 pass 理由 — Bot/Bundler 高也可能是做市商在铺流动性
2. Bot 率突然下降比 Bot 率高更危险 — 说明做市商撤了
3. TG/Discord 刷屏越快越危险 — 大户不在里面聊天
4. CEX 上币往往是顶部 — 只有 Binance 现货是例外
5. "涨太多了"不是合理判断 — 用注意力定价，不用价格否定叙事
6. 负面 FUD 可能是买入信号 — 洗纸手 + 筹码集中到信仰者
7. 英文 CT → 中文 CT 传播时，英文 CT 早期玩家通常在考虑出货
8. 越"傻"的 meme 越容易传播 — 0.5秒能get到的梗比需要思考3秒的梗传播力强10倍
9. 争议是传播燃料 — 有对立面的叙事比和谐的叙事传播快3-5倍
10. 内盘看速度，外盘看叙事 — 不要在错误的阶段用错误的逻辑
